    OVERVIEW

    Mint Paper에서 주최하는 페스티벌 ‘Have a nice trip 2023’를 중심으로
    리디자인. 페스티벌, 음악, 여행을 컨셉으로 트로피컬 포인트의 컬러를 사용하고
    일러스트와 영상을 직접 제작한 동적 구현 웹사이트

    담당 업무 전체 레이아웃 구성
    디자인 가이드라인 구축 (폰트 스타일, 타이틀 및 본문 텍스트 사이즈, 마진 등)
    메인 페이지 timetable / footer 디자인 퍼블리싱 및 스크립트 효과 작업

    사용 기술 HTML5 / CSS3 / Javascript / JQuery / Figma / Github organization

    주요 스크립트

    ScrollTrigger를 이용한 스크롤 이벤트

    3면의 배경에 scrollTrigger를 활용한 스크롤 이벤트.
    scale, start지점과 속도를 다르게 지정하여 각각 다른 애니메이션으로
    입체감을 살리고, 흐르는 글자와 함께 이동해 페스티벌의 분위기에 맞게
    역동적이며 화려한 느낌을 주었다.

    ScrollTrigger 기술서 보기

    Swiper 사용

    autoplay : delay 2500을 주어 자동으로 롤링. loop : true 설정하여 무한히 돌아가는 루프 설정

    addClass(), removeClass()

    스와이퍼 가수 클릭 시 상단 가수 내용 자동 교체.
    스크립트 addClass(‘active’) .on('click', () =>{ addClass(), removeClass() })
    가수 클릭 시 나머지 목록과 번갈아 적용

    scrollTo

    offset top 부분을 선언하여 클릭시 scrollTo 박스의 offset top으로 이동 시킴.
    behavior: "smooth" 설정하여 부드러운 이동.

    스크롤에 따른 SVG Animation

    HTML에 svg를 작성. css 스타일 값을 지정, js에 스크롤 시 경로에 따른 움직임.
    스크롤 이벤트가 발생할 때마다 scrolled 함수가 호출되면서 svg 경로 애니매이션이
    구현되도록 작성.

    마우스 휠 이벤트

    스크롤바 기본 성격 제거, nextElementSibling, prevElementSibling 이용하여
    휠을 내렸을때 각 박스영역의 top에서 멈추도록하여 마우스 휠을 이용한 스크롤

    ILLUSTRATION

    페스티벌의 즐거운 컨셉을 효과적으로 전달하기 위해
    비비드 색감으로 작업한 일러스트레이션과
    작업물을 바탕으로 영상 제작.

    그래픽 요소와 컬러에서 컨셉의 일관성을 유지시키며
    전체적으로 동적인 느낌을 많이 주어
    화려한 페이지를 구현하였다.

    프로젝트 결과

    메인페이지와 서브페이지 예정대로 완성
    스크립트 효과 부분은 레퍼런스 사이트보다 부족한 적용
    일부 디자인의 디테일 부족

    부족한 점

    Github 활용의 부족 팀 작업을 하면서 Github의 Branch나 merge 등을 이번 기회에 활용을 많이 해보고 싶었으나 사용하지 못했다.
    다른 팀원이 push를 하면 다시 매번 pull을 하여 받아야 하는 번거로운 과정을 반복하는 등 불편함이 있었고,
    더 좋은 시스템 을 잘 활용하지 못하였다.

    잘한 점

    업무 분담의 명확함 디테일 요소가 많은 작업이어서 시간 분배가 중요했다.
    네 명의 팀원이 두 명씩 나뉘어 일러스트 영역과 편집의 영역으로 확실한 업무 분담을 하였고,
    일정에 맞춰 작업을 진행 함으로써 작업 요소가 많음에도 불구하고 시간 내에 프로젝트를 완료할 수 있었다.

    배운 점

    ScrollTrigger 사용 스크립트 적용시 스크롤 이벤트 부분을 담당.
    스크롤 트리거의 기본적인 사용법을 이해하고 적용시키는 방법을 배웠다.
    아직 사용해보지 못한 속성과 콜백함수 등 함께 적용하면 무수히 다양한 효과를 구현할 수 있을거라 생각된다.

    REVIEW

    HTML과 CSS, JS의 연관성 HTML과 CSS, JS 모두 연결되어 있다는 것. 특히 스크립트 효과를 주면서 HTML 구조를 변경해야 할 때가 있었고,
    처음 HTML 구조를 짤 때 생각하지 않으면 하위 영역에 영향을 주어 이후 수정이 굉장히 어려워진다는 점. HTML
    작업을 시작할 때부터 어떤 효과를 어떻게 적용할 것 인지 생각하고 시작해야 된다는 점을 배웠다.

    퍼블리싱 가이드라인의 중요성 디자인의 특성상 메인 페이지에 3명이 동시 작업을 진행하게 되었다. 그러면서 다른 팀 원의 HTML과 CSS 구조를
    바꿔야 하는 부분이 있었는데 지정된 가이드라인에 맞춰 작업 해 놓아 내가 작업하지 않은 부분도 알아보기 쉬웠고
    수정을 하더라도 크게 문제없었다. 만약 가이드라인 없이 개별적으로 각자 시작했다면 작업이 원활하지 못했을 것이
    고, 시간이 부족하더라도 이런 절차가 있어야 이후 작업 과정에 시간을 단축시킬 수 있다는 생각이 든다.
